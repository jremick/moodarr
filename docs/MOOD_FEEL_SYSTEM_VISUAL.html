Moodarr algorithm documentation

Mood/Feel Profile System Map

Moodarr is trying to learn what a user means by mood words like "cozy", "dark", "weird", and "light". The system starts with a generic Feel Space for the catalog, then learns small user-specific translations on top of that baseline.

The Core Idea

Generic search knows the broad English meaning of a word. Moodarr should learn the user's private meaning of that word for watch selection, while keeping catalog facts exact.

Local-first profile Hard facts stay deterministic AI can rerank known candidates only Feedback is gradual and resettable Reliability-gated learning Profile versions are stamped Evals before quality claims
02

Complete System Flow

The system is a pipeline, not a single model. Personalization is one bounded layer in the middle of deterministic retrieval and scoring.

Catalog Truth

Plex and Seerr/Jellyseerr provide titles, availability, request status, posters, IDs, ratings, genres, runtime, and summaries.

Baseline Feel Space

Each title becomes a feature document with mood terms, tone terms, watchability terms, local vectors, FTS text, and mood feature scores.

User Query

Intent parsing extracts hard filters, soft mood words, reference titles, availability intent, and watch context.

Feel Profile Translation

If the query contains a calibrated term, learned feature weights translate that word into the user's private meaning.

Hybrid Retrieval

Broad candidate generation combines catalog scan, FTS, semantic vectors, mood index, reference neighborhoods, and Seerr augmentation.

Scoring

Independent score buckets rank known candidates: query, semantic, mood, reference, taste, preference, profile, feedback, quality, friction, availability, novelty.

Diversity

A deterministic pass keeps broad mood prompts from collapsing into near-duplicates.

Optional AI Rerank

AI can reorder safe known candidates and explain tradeoffs, but cannot invent titles or override availability.

Frictionless Feedback

More-like, less-like, hide, right-mood, wrong-mood, pairwise picks, opens, saves, and future swipes are stored as structured signals.

Profile Update + Evals

Reliable mood actions nudge term weights, update profile versions, respect same-session caps, and create holdouts. Evals verify lift, journeys, replay, and drift before real user data arrives.

03

Baseline Vs Personalized Meaning

Baseline Feel Space

The general English/catalog meaning shared by every user. It answers, "What does this title generally feel like?"

mood:cozy mood:weird tone:suspenseful watch:low-commitment runtime:short movie
+

Feel Profile Delta

The user's learned translation for recurring words. It answers, "When this user says this word, which features should move up or down?"

cozy -> witty comedy cozy -> magical adventure dark -> tense mystery dark -> horror intensity light -> easy background
04

What Happens During One Search

1

Parse the request

"Dark movie" becomes a movie filter plus soft mood terms. Hard constraints such as movie, runtime, availability, and exclusions are deterministic.

2

Load the profile

The engine loads the learned profile for the active watch context: currently solo or group, and records the profile version used for the slate.

3

Translate calibrated terms

If the profile contains dark, the system builds a weighted adjustment such as thriller up, grounded up, horror down.

4

Retrieve broadly

The engine keeps recall high using local catalog data, FTS, semantic vectors, mood feature scores, and requestable Seerr candidates when useful.

5

Score with a bounded profile bucket

The profile bucket is centered at 50. Strong personal matches add a limited positive delta; mismatches subtract a limited delta.

6

Show results and collect signals

Normal actions such as more-like, less-like, hide, and right-mood become structured events. Reliability class decides whether a signal can train mood-term meaning.

Current Algorithms In Plain English

Feature Builder

Turns catalog metadata into mood, tone, watchability, runtime, rating, genre, and semantic features.

Intent Parser

Splits user text into hard filters and soft signals. Recent hardening promotes dark, tense, intense, and suspenseful into mood language.

Profile Translator

Matches calibrated terms in the query and creates feature adjustments scaled by confidence, effective evidence, and conflict.

Hybrid Retriever

Prioritizes recall. It gathers candidates from several channels before scoring and optional AI reranking.

Deterministic Scorer

Combines independent buckets. The profile score is visible and bounded, so personalization can move close candidates without becoming the whole ranker.

Learning Update

Only medium/high reliability actions update term weights. Reason chips add targeted deltas, reliability controls strength, conflicts shrink confidence, and repeated same-session term updates are capped.

Replay + Rollback

Holdouts and checkpoints let the system compare profile versions. Drift alerts flag conflicted terms, and rollback restores one term/context as a new version while preserving history.

08

Robustness Assessment

Strong Now

  • Hard filters and availability are deterministic gates.
  • Profile learning is local, bounded, inspectable, and resettable.
  • Weak and diagnostic actions cannot train term-profile meaning.
  • Cold-start and contradictory term evidence has reduced profile influence.
  • Checkpoint timeline diagnostics, drift alerts, scoped rollback, and replay compaction are in place.
  • AI reranking cannot invent unknown titles or override catalog state.
  • Synthetic, adversarial, and multi-event journey evals now test multiple meanings, weak actions, pairwise choices, context isolation, and fragile language classes.
  • MovieLens Tag Genome can be validated locally as aggregate coverage without writing DB rows or committed derived data.

Still Fragile

  • The baseline Feel Space is still mostly deterministic metadata extraction.
  • Synthetic profiles are hand-authored and should not be mistaken for real behavior.
  • Cold start has limited signal until repeated terms receive explicit feedback.
  • Replay and drift diagnostics are structurally ready, but need real held-out usage to become more predictive.

Next Hardening

  • Refine rollback UX after local usage shows common drift patterns.
  • Run local-only external mood/tag validation where license and provenance are acceptable.
  • Add mobile/iOS signal collection after web/backend semantics settle.
