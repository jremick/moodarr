# Feelerr / Moodarr Project Instructions

Version: 1.0.0
Last updated: 2026-06-15

## Design System

- For any UI, UX, or frontend design work in this app, use `docs/design/opus-design-system.html` as the project design-system source of truth.
- Treat `docs/design/opus-admin-mockup.html` as the approved admin redesign direction unless the user supersedes it.
- Keep implementation refinements inside the Screening Desk system: warm catalog-console surfaces, existing CSS tokens, compact operational panels, Phosphor-style icon language, and restrained admin density.
- Do not introduce a detached visual system for new app surfaces without explicit approval.
